# Campaign Image Generation App
I need a small application that will help generate campaign ideas for social media. The backend is going to be written in CrewAI while the frontend is going to be a simple streamlit app. Pydantic will be used to assure needed information is retrieved from LLM interaction.

It needs to be able to take a campaign brief which will be in a JSON format (JSON only, no YAML) or supplied via the frontend through a form. It will also need to have the ability to upload existing images.

**The Campaign Brief**
* Product(s) – at least two different products
* Target region/market
* Target audience
* Campaign message

**key features**
* Storage will be local
* each campaign will be complete when it has three*  images in these ratios 1:1, 9:16, 16:9
* images can be uploaded from the frontend
* campaigns can be shared via a link in the upper right
* campaigns can be approved
* if a database is needed then a flat database.json will maintained in place of a database.

# Frontend
This is going to be a streamlit app. It will he a single page.

## Left Nav:
* Generate Campaign +
* Previously Generated Campaigns

The button to generate a new form will always be at the top. The campaigns will be tracked in the database after they're generated and will be displayed here by name. Clicking on a campaign name will bring up the campaign display (Previously Generated Campaigns)

## Generate Campaign Form:
A name will need to be entered for the campaign.
At the top of the page there will a place to upload a JSON file (JSON only) which should contain the campaign brief details from above. This area will have a button that will swap the upload to a manual entry form with the following fields:
* Products: A text input that is comma separated and must contain at least two different products
* Target Region/Market: A text input
* Target Audience: A text input
* Campaign Message: A text input

Validation will be done on the input fields and shown below each input field in a subtle red banner. All fields are required.

The campaign brief will be stored in the local database.

In the middle of the page there will be a place to upload images. The images will be displayed in a grid. They will be stored in the local file structure according to their aspect ratio.

At the bottom of the page there will be a button to generate the campaign. Once this is clicked the page will be locked behind a shroud until the request comes back successfully, the word "Generating" with a loading type animation will be displayed centered on the page. Then the user will be taken to the campaign display page for that campaign.

## Previously generated campaigns:
This will be a display of the campaign brief followed by the three generated images in a grid.

In the upper right hand corner there will be a button to share the campaign along with a button to approve the campaign.

The name of the campaign will be displayed at the top of the page as an H2.

Below the images grid there will be sections with an H3 header and then well formatted paragraphs for the branding rpeort and future campaigns. Then there will be a collapsed sections for the branding details and marketing details.

Share link: This means that the URL for current campaign will be a URL safe version of the name.

Approved button: The state of this button will come from the database. A campaign will have an approved state of false by default. The button says "Needs Approval", when the valuue is false and is red. When the value is true the button says "Approved" and is green. The state is toggled by clicking the button.

# Backend
This is going to be a CrewAI app. This should be configured to use Gemini Pro for both image extraction (e.g., logo detection) and image generation.

## Database models
The database will be a flat json file called database.json. It will be accessed by the backend using the python json module. It will be in a separate class/file so that it can be swapped out later and have a simple API with add, update, delete, get.

**Campaign:**
* id (int)
* name (array)
* products (array)
* target_region (string)
* target_audience (string)
* campaign_message (string)
* language (string: default US_en)
* approved (boolean: default false)
* initialImages: (array)
* generatedImages: (array[Image])
* brandingDetails: (string)
* brandingReport: (string)
* marketingDetails: (string)
* futureCampaigns: (string)
* logoImage: (Image)

**Image:**
* aspectRatio (string)
* path (string)

## When a campaign form is submitted
The campaign brief is stored in the database, and ID for the campaign is generated. The ID is generated by incrementing the highest ID in the database.
The images are stored in the local file structure according to their aspect ratio, if the image does not meet one of the three aspect ratios then it is stored in a "general" folder. These submitted images will be have their path stored in the initialImages field of the campaign model.

Then a crewAI flow will be triggered to generate the campaign images that do not already exist. This is determined by the aspect ratio of the submitted images.
#### Step 1
A branding detail will need to be extracted using the "branding extract" agent; to be stored in the `brandingDetails` field on the campaign. The logo will also need to be extracted from the given image so that it can be provided to the image generation model later (not composited locally). Also a campaign marketing detail will need to be extracted using the "marketing extract" agent; to be stored in the `marketingDetails` field and the `language` field of the campaign. A "logo extract" agent will be needed to lift the current logo out of the given image so that it can be used during the image generation step (Gemini Pro input). The logo will be stored locally and the path will be stored in the `logoImage` field of the campaign.
#### Step 2
The flow will then continue to generate the campaign images with the branding and marketing details passed through an "image summary" agent for the missing image ratios. The logo path and campaign message will be provided to Gemini Pro as part of the generation call, rather than composited with Pillow. The generated images will be stored in the `generatedImages` field of the campaign.
#### Step 3
In the next section of the flow a "branding report" agent will be used to evaluate how well the generated images match the branding details and give any potential feedback which will be stored in the `brandingReport` field of the campaign.
A "future campaigns" agent will be used to generate campaign ideas for the next 3 months. Relevant tools to complete these tasks need to be created and stored in the `futureCampaigns` field of the campaign.
#### Step 4
The flow will then complete and the campaign will be stored in the database. The generated ID is returned to the frontend so that the campaign details can be displayed.

### Agents
#### branding extract:
- role: Branding and product marketing expert
- goal: To evaluate a given image, target audience and campaign message to create a summary to be used in image generation that determines details about the logo, branding, and relevant product information. And to determine what should and what should not be included in the image.
- backstory: You are a product branding expert with 25 years of experience in successful marketing campaigns.

#### logo extract:
- role: Logo extraction expert
- goal: To lift the logo from a given image.

#### marketing extract:
- role: Global Marketing Researcher
- goal: To evaluate a given region, target audience and campaign message to create a summary to be used in image generation that determines details about the language of the region, current trends for the target audience, and to customize an appropriate marketing message with professional guardrails in mind. Along with focusing the campaign on performance and trying to engage new audiences.
- backstory: You are a global marketing researcher, intune with localized social trends and what makes a successful ad campaign for ${targetAudiend} in ${targetRegion}

#### image summary:
- role: Image generation expert
- goal: To evaluate a given image, branding details and marketing details to create a summary to be used in image generation that determines what should and what should not be included in the image.

#### branding report:
- role: Branding and product marketing expert
- goal: To evaluate a given image and branding details and produce a report scoring how closely the two align and details about what is working and what could be improved.
- backstory: You are a product branding expert with 25 years of experience in successful marketing campaigns.

#### future campaigns:
- role: Marketing Manager
- goal: To look at the given campaign brief and determine 3 future campaign ideas to be delivered on a monthly basis talking about the long term stratgey and why you think it will be an efficient next step given the target audience and target region. The future campaigns need to include campaign briefs of their own.
- backstory: You are a marketing manager specializing in multi-month campaigns.

### When campaign details are requested
The campaign details are returned to the frontend based on the ID. In this version the entire campaign will be sent back, except the images will need to have their path updated to be relative to the local file structure to be displayed on the frontend.

### When approval is toggled
A simple API will find the record by ID in the database and toggle the approved field. A success will be returned to the frontend.

## Python Classes
Database: This will be a simple class that will wrap interaction with the database.json file to streamline updating the API and leaving way for future implementation with different databases.

Image: This take a path and will be able to return a frontend or backend path. It will also be used to determine if an image already exists in the local file structure.

ImageGenerator: This will be a wrapper class for the LLM when generating an image and extracting the logo in case it needs to be changed out in the future.
